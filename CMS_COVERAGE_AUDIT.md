# 第三阶段后台驱动覆盖审计

## 已完全后台驱动

- 站点基础信息：`siteName`、品牌色、Logo 文案、联系信息、默认 SEO、`featuredLinks`、`footerGroups`、`socialLinks`、`legalText`
- 主导航：`/admin/site` 可维护导航名称、链接、排序与增删
- 页面内容：`/admin/pages` 可编辑页面标题、slug、SEO、block 结构与 block 字段
- 产品详情扩展：可从产品详情模板复制生成新产品页，再通过后台保存进入正式前台
- 媒体资源：`/admin/media` 可上传、删除和复用文件，前台通过 `/uploads/...` 统一读取
- 前台保存联动：站点设置与页面保存均写入 D1，媒体文件写入 R2 或本地 uploads，并被正式前台读取

## 部分后台驱动

- 新闻中心与扩展页内容：当前已作为正式页面记录进入后台，可编辑 block 与 SEO，但尚未提供专门的“新闻列表 / 新闻详情”结构化字段
- 产品矩阵：当前由独立产品详情页驱动并自动回流 `product-feed`，但没有额外的产品分类、标签、参数表专用字段

## 仍需补字段或入口

- 多语言独立内容维护：当前项目以前台中文主站为正式交付对象，`cn/` 镜像页未纳入本轮后台正式维护范围
- 新闻发布日期、作者、标签等结构化元数据
- 下载页、APP 页的专用文件上传字段与版本信息字段
- 招聘、供应商等低优先级企业流程页的专用后台入口

## 第三阶段结论口径

- 后台已经能够驱动当前正式前台的主要站点设置、页面内容与媒体资源
- 普通运营人员不需要手改 JSON 即可维护当前正式交付范围内的大部分前台内容
- 后续若继续扩展招聘、下载中心、新闻元数据等领域，应在不改变三个基准页骨架的前提下补专用 schema 与表单
